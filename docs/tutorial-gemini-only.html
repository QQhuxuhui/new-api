Google Gemini CLI 使用教程

跟着这个教程，轻松在你的电脑上安装并使用 Gemini CLI

0

前置要求

系统要求：

  • Node.js 18+（推荐 Node 20+）
  • 支持 Windows、macOS 和 Linux

检查环境：

node --version
💡 提示： 如果版本低于 18，请先升级 Node.js。Gemini CLI 需要 Node.js 18 及以上版本。
1

安装 Node.js 环境

Gemini CLI 需要 Node.js 环境才能运行。请根据你的操作系统选择安装方式：

🪟 Windows

方法一：官网下载（推荐）

  1. 打开浏览器访问 nodejs.org
  2. 点击 "LTS" 版本进行下载（推荐长期支持版本）
  3. 下载完成后双击 .msi 文件
  4. 按照安装向导完成安装，保持默认设置即可

方法二：使用包管理器

# 使用 Chocolatey
choco install nodejs

# 或使用 Scoop
scoop install nodejs
💡 Windows 注意事项：
  • 建议使用 PowerShell 而不是 CMD
  • 如果遇到权限问题，尝试以管理员身份运行
🍎 macOS

方法一：使用 Homebrew（推荐）

# 更新 Homebrew
brew update

# 安装 Node.js
brew install node

方法二：官网下载

  1. 访问 nodejs.org
  2. 下载适合 macOS 的 LTS 版本
  3. 打开下载的 .pkg 文件，按照安装程序指引完成安装
💡 macOS 注意事项：
  • 如果遇到权限问题，可能需要使用 sudo
  • 首次运行可能需要在系统偏好设置中允许
  • 建议使用 Terminal 或 iTerm2
🐧 Linux / WSL2

方法一：使用 NodeSource 仓库（推荐）

# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs

方法二：使用系统包管理器

# Ubuntu/Debian
sudo apt update
sudo apt install nodejs npm

# CentOS/RHEL/Fedora
sudo dnf install nodejs npm
💡 Linux 注意事项：
  • 某些发行版可能需要安装额外的依赖
  • 如果遇到权限问题，使用 sudo
  • 确保你的用户在 npm 的全局目录有写权限

验证安装：

node --version
npm --version

如果显示版本号，说明安装成功了！

2

安装 Gemini CLI

全局安装命令：

# Windows/macOS/Linux 通用
npm install -g @google/gemini-cli
⚠️ 权限提示：
  • Windows：请使用管理员权限运行 PowerShell
  • macOS/Linux：如遇报错，请尝试 sudo npm install -g @google/gemini-cli

验证安装：

gemini --version

如果显示版本号，说明 Gemini CLI 已经成功安装了。

3

配置中转 API 环境变量

为了让 Gemini CLI 连接到你的中转服务，需要设置环境变量：

🪟 Windows (PowerShell)

临时设置（当前会话）：

$env:GOOGLE_GEMINI_BASE_URL = "{{BASE_URL}}"
$env:GEMINI_API_KEY = "你的API密钥"
$env:GEMINI_MODEL = "gemini-2.5-pro"

永久设置（用户级）：

# 设置用户级环境变量（永久生效）
[System.Environment]::SetEnvironmentVariable("GOOGLE_GEMINI_BASE_URL", "{{BASE_URL}}", [System.EnvironmentVariableTarget]::User)
[System.Environment]::SetEnvironmentVariable("GEMINI_API_KEY", "你的API密钥", [System.EnvironmentVariableTarget]::User)
[System.Environment]::SetEnvironmentVariable("GEMINI_MODEL", "gemini-2.5-pro", [System.EnvironmentVariableTarget]::User)

💡 设置后需要重新打开 PowerShell 窗口才能生效。

验证设置：

# PowerShell
echo $env:GOOGLE_GEMINI_BASE_URL
echo $env:GEMINI_API_KEY
echo $env:GEMINI_MODEL
🍎 macOS

临时设置（当前会话）：

export GOOGLE_GEMINI_BASE_URL="{{BASE_URL}}"
export GEMINI_API_KEY="你的API密钥"
export GEMINI_MODEL="gemini-2.5-pro"

永久设置：

# 对于 zsh（macOS 默认）
echo 'export GOOGLE_GEMINI_BASE_URL="{{BASE_URL}}"' >> ~/.zshrc
echo 'export GEMINI_API_KEY="你的API密钥"' >> ~/.zshrc
echo 'export GEMINI_MODEL="gemini-2.5-pro"' >> ~/.zshrc
source ~/.zshrc

# 对于 bash
echo 'export GOOGLE_GEMINI_BASE_URL="{{BASE_URL}}"' >> ~/.bash_profile
echo 'export GEMINI_API_KEY="你的API密钥"' >> ~/.bash_profile
echo 'export GEMINI_MODEL="gemini-2.5-pro"' >> ~/.bash_profile
source ~/.bash_profile

💡 验证：echo $GOOGLE_GEMINI_BASE_URL

🐧 Linux / WSL2

临时设置（当前会话）：

export GOOGLE_GEMINI_BASE_URL="{{BASE_URL}}"
export GEMINI_API_KEY="你的API密钥"
export GEMINI_MODEL="gemini-2.5-pro"

永久设置：

# 对于 bash（Linux 默认）
echo 'export GOOGLE_GEMINI_BASE_URL="{{BASE_URL}}"' >> ~/.bashrc
echo 'export GEMINI_API_KEY="你的API密钥"' >> ~/.bashrc
echo 'export GEMINI_MODEL="gemini-2.5-pro"' >> ~/.bashrc
source ~/.bashrc

# 对于 zsh
echo 'export GOOGLE_GEMINI_BASE_URL="{{BASE_URL}}"' >> ~/.zshrc
echo 'export GEMINI_API_KEY="你的API密钥"' >> ~/.zshrc
echo 'export GEMINI_MODEL="gemini-2.5-pro"' >> ~/.zshrc
source ~/.zshrc

💡 验证：echo $GOOGLE_GEMINI_BASE_URL

📝 配置要点：
  • API Base URL：{{BASE_URL}}
  • API Key：将 你的API密钥 替换为你的实际密钥
  • 模型：GEMINI_MODEL 可根据需要更改为其他 Gemini 模型
  • 注意：必须同时设置 GEMINI_API_KEY 和 GOOGLE_GEMINI_BASE_URL 两个环境变量
4

开始使用 Gemini CLI

交互模式：

gemini

直接输入 gemini 即可进入交互式对话模式。

单次提问：

gemini "Explain quantum computing briefly"

管道输入：

cat file.txt | gemini "Summarize this file"

在特定项目中使用：

# 进入你的项目目录
cd /path/to/your/project

# 启动 Gemini CLI
gemini

常见问题解决

安装时提示 "permission denied" 错误

  • Windows：以管理员身份运行 PowerShell，或配置 npm 使用用户目录：npm config set prefix %APPDATA%\npm
  • macOS/Linux：使用 sudo npm install -g ...，或配置 npm 使用用户目录：npm config set prefix ~/.npm-global 并添加到 PATH

PowerShell 执行策略错误（Windows）

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

环境变量设置后不生效

  • Windows：重新打开 PowerShell 或注销重新登录
  • macOS：确认修改了正确的文件（.zshrc 或 .bash_profile），并重新打开 Terminal
  • Linux：确认修改了正确的文件（.bashrc 或 .zshrc），并执行 source 或重启终端

Node.js 版本过低

Gemini CLI 要求 Node.js 18+，如果版本过低请升级：

# macOS (Homebrew)
brew upgrade node

# Linux (NodeSource)
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
