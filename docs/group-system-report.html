1. 系统概览

🎯 设计目标

分组系统是 New-API 的核心功能模块，用于实现资源隔离、权限控制、倍率管理。层级分组系统在原有基础上引入了父子级关系，使分组管理更加灵活和高效。

架构设计

核心组件：

  • GroupRatio：分组基础倍率配置
  • GroupGroupRatio：用户分组对使用分组的模型倍率
  • GroupTree：层级树结构（父级→子级映射）
  • ChildToParent：反向索引（子级→父级映射）

关键特性

特性 说明 优势
层级管理 支持父级分组自动展开为子级分组 简化配置，减少重复工作
倍率回退 子级→父级→默认值的倍率查找链 灵活配置，支持继承
自动过滤 用户界面只显示父级和独立分组 简化用户操作，避免混淆
自动展开 渠道缓存自动展开父级分组 确保路由正确，无需手动维护
交集验证 令牌分组与套餐分组的智能交集 支持父级分组，灵活权限控制

✅ 向后兼容

未配置 GroupTree 时，系统保持原有行为。所有分组都是独立分组，不会影响现有功能。

2. 核心概念

父级分组

定义：在 GroupTree 中作为 key 存在的分组

特点：

  • 可以包含多个子级分组
  • 在令牌创建时显示
  • 选择时自动授予所有子级权限
  • 渠道配置时自动展开为子级

示例：claude, openai

子级分组

定义：在 GroupTree 中作为 value 存在的分组

特点：

  • 属于某个父级分组
  • 在令牌创建时隐藏
  • 用作渠道缓存的 key
  • 可以继承父级的倍率配置

示例：claude-包月, claude-按量, openai-包月, openai-按量

独立分组

定义：既不是父级也不是子级的分组

特点：

  • 在 GroupTree 中不存在
  • 在令牌创建时显示
  • 保持原有行为
  • 无继承关系

示例：default, gemini, glm

GroupTree 配置示例

claude (父级分组) ├── claude-包月 (子级分组) └── claude-按量 (子级分组) openai (父级分组) ├── openai-包月 (子级分组) └── openai-按量 (子级分组) default (独立分组) gemini (独立分组) glm (独立分组)

JSON 配置格式

{
  "claude": ["claude-包月", "claude-按量"],
  "openai": ["openai-包月", "openai-按量"]
}

3. 数据结构

3.1 GroupRatio（分组基础倍率）

类型：map[string]float64

用途：定义每个分组的基础计费倍率

读写锁：groupRatioMutex sync.RWMutex

// 示例配置
{
  "default": 1.0,
  "vip": 0.9,
  "svip": 0.8,
  "claude": 1.2,
  "claude-包月": 1.0,
  "claude-按量": 1.5
}

3.2 GroupGroupRatio（分组间模型倍率）

类型：map[string]map[string]float64

用途：定义用户分组对使用分组的模型倍率

读写锁：groupGroupRatioMutex sync.RWMutex

// 示例配置
{
  "vip": {
    "claude-按量": 0.9,
    "openai-按量": 0.85
  },
  "svip": {
    "claude-按量": 0.8,
    "openai-按量": 0.75
  }
}

3.3 GroupTree（层级树结构）

类型：map[string][]string

用途：定义父级分组到子级分组的映射

读写锁：groupTreeMutex sync.RWMutex

// 示例配置
{
  "claude": ["claude-包月", "claude-按量"],
  "openai": ["openai-包月", "openai-按量"]
}

3.4 ChildToParent（反向索引）

类型：map[string]string

用途：快速查找子级分组的父级分组

读写锁：childToParentMutex sync.RWMutex

// 自动生成的反向索引
{
  "claude-包月": "claude",
  "claude-按量": "claude",
  "openai-包月": "openai",
  "openai-按量": "openai"
}

⚠️ 线程安全

所有数据结构都使用读写锁保护，支持并发读取和独占写入。更新操作会原子性地更新 GroupTree 和 ChildToParent。

4. 核心函数

4.1 GroupTree 查询函数

函数 签名 功能
IsParentGroup func(group string) bool 判断分组是否为父级分组
GetChildGroups func(parent string) []string 获取父级分组的所有子级分组
GetParentGroup func(child string) string 获取子级分组的父级分组
ExpandGroup func(group string) []string 展开分组（父级→子级列表，其他→自身）
GetAllParentGroups func() []string 获取所有父级分组名称
GetAllChildGroups func() []string 获取所有子级分组名称

4.2 分组过滤函数

函数 签名 功能
GetIndependentGroups func(allGroups []string) []string 获取独立分组（非父级非子级）
GetUserSelectableGroups func(allGroups []string) []string 获取可选分组（父级+独立，排除子级）

4.3 倍率查询函数

GetGroupRatio(name string) float64

倍率回退逻辑：

  1. 优先查找子级分组的倍率
  2. 未找到则查找父级分组的倍率
  3. 都未找到则返回默认值 1.0
// 示例：查找 "claude-按量" 的倍率
func GetGroupRatio(name string) float64 {
    // 1. 先查 "claude-按量" 的倍率
    if ratio, ok := groupRatio["claude-按量"]; ok {
        return ratio // 返回 1.5
    }

    // 2. 查找父级 "claude" 的倍率
    parent := GetParentGroup("claude-按量") // 返回 "claude"
    if parent != "" {
        if ratio, ok := groupRatio["claude"]; ok {
            return ratio // 如果配置了，返回 1.2
        }
    }

    // 3. 返回默认值
    return 1.0
}

GetGroupGroupRatio(userGroup, usingGroup string) (float64, bool)

模型倍率回退逻辑：

  1. 优先查找用户分组对子级分组的模型倍率
  2. 未找到则查找用户分组对父级分组的模型倍率
  3. 都未找到则返回 (-1, false)

4.4 配置管理函数

函数 功能 说明
UpdateGroupTreeByJSONString 更新 GroupTree 配置 自动构建反向索引，原子性更新
ValidateGroupTree 校验 GroupTree 配置 检查重复子级、嵌套层级、自引用
CheckGroupTree 校验 JSON 字符串 解析并校验配置格式

⚠️ 配置校验规则

  • 禁止重复子级：同一个子级不能属于多个父级
  • 禁止嵌套层级：子级不能同时也是父级
  • 禁止自引用：父级和子级不能同名

5. 业务流程

5.1 令牌创建流程

步骤 1：获取所有分组
从 GroupRatio 获取所有分组名称
步骤 2：过滤子级分组
调用 GetUserSelectableGroups() 过滤出父级和独立分组
步骤 3：构建响应数据
为父级分组附加 children 信息，方便前端展示
步骤 4：前端显示
用户只看到父级分组和独立分组，选择父级时自动授予所有子级权限

示例：用户创建令牌时看到的分组选项

❌ 不使用 GroupTree（混乱）

  • claude-包月
  • claude-按量
  • openai-包月
  • openai-按量
  • default
  • gemini

用户需要理解细分，选择繁琐

✅ 使用 GroupTree（简洁）

  • claude (包含: claude-包月, claude-按量)
  • openai (包含: openai-包月, openai-按量)
  • default
  • gemini

用户只需选择大类，自动授权子级

5.2 渠道路由流程

步骤 1：套餐选择
用户使用令牌调用 API，系统为用户选择套餐（如果启用套餐系统）
步骤 2：获取套餐分组
从 UserPlan 获取 channel_groups（可能包含父级分组）
步骤 3：展开父级分组
遍历 channel_groups，调用 ExpandGroup() 展开父级分组为子级分组
步骤 4：令牌分组交集
如果令牌指定了分组，计算令牌分组与套餐分组的交集（支持父级展开）
步骤 5：渠道选择
从展开后的分组中，使用 CacheGetRandomSatisfiedChannel() 选择渠道
步骤 6：路由成功
请求被路由到选中的渠道，记录使用的实际子级分组

🔍 关键代码位置

  • middleware/distributor.go:158-170 - 套餐分组展开
  • middleware/distributor.go:174-220 - 令牌分组交集验证

5.3 套餐选择流程（交集逻辑）

场景：令牌分组为 "claude"，套餐分组为 ["claude-按量", "openai-包月"]

// 伪代码
tokenGroup := "claude" // 令牌配置的分组
planGroups := ["claude-按量", "openai-包月"] // 套餐配置的分组

// 1. 检查令牌分组是否为父级
if IsParentGroup(tokenGroup) {
    // 2. 展开为子级列表
    tokenChildren := GetChildGroups(tokenGroup)
    // tokenChildren = ["claude-包月", "claude-按量"]

    // 3. 与套餐分组求交集
    planGroupSet := map["claude-按量": true, "openai-包月": true]
    effectiveGroups := []
    for _, child := range tokenChildren {
        if planGroupSet[child] {
            effectiveGroups = append(effectiveGroups, child)
        }
    }
    // effectiveGroups = ["claude-按量"]

    // 4. 使用交集结果作为实际可用分组
    channelGroups = effectiveGroups
} else {
    // 令牌是子级或独立分组，直接匹配
    ...
}

// 5. 从 channelGroups 中选择渠道
channel := CacheGetRandomSatisfiedChannel(channelGroups[0], modelName)

❌ 旧逻辑（直接匹配）

令牌分组: claude

套餐分组: ["claude-按量"]

结果：匹配失败，拒绝请求

✅ 新逻辑（父级展开）

令牌分组: claude → 展开为 ["claude-包月", "claude-按量"]

套餐分组: ["claude-按量"]

结果：交集 = ["claude-按量"]，匹配成功

5.4 故障切换流程

步骤 1：当前套餐所有渠道不可用
用户请求失败，所有优先级的渠道都尝试过
步骤 2：检查是否启用 auto_switch
判断当前 UserPlan 的 auto_switch 是否为 1
步骤 3：获取候选套餐
调用 GetFailoverCandidates() 获取其他可用套餐（按优先级排序）
步骤 4：遍历候选套餐
对每个候选套餐，展开其 channel_groups（父级→子级）
步骤 5：令牌分组验证
如果令牌有分组约束，计算交集（支持父级展开）
步骤 6：尝试选择渠道
在候选套餐的分组中尝试选择渠道
步骤 7：切换套餐
找到可用渠道后，调用 SwitchToUserPlan() 切换套餐，更新上下文

🔍 关键代码位置

  • service/plan_failover.go:106-119 - 候选套餐分组展开
  • service/plan_failover.go:128-162 - 令牌分组交集验证
  • service/plan_failover.go:361-374 - 成功切换后的分组更新

5.5 倍率计算流程

步骤 1：获取使用分组
从上下文获取实际使用的分组（如 "claude-按量"）
步骤 2：查找模型倍率
调用 GetGroupGroupRatio(userGroup, usingGroup)
步骤 3：模型倍率回退
先查子级，未找到则查父级，都未找到返回 -1
步骤 4：查找分组倍率
如果模型倍率未找到，调用 GetGroupRatio(usingGroup)
步骤 5：分组倍率回退
先查子级，未找到则查父级，都未找到返回 1.0
步骤 6：计算最终费用
基础价格 × 模型倍率 × 分组倍率 × 渠道倍率 = 最终费用

示例：计算用户使用 "claude-按量" 的费用

// 配置
GroupGroupRatio = {
  "vip": {
    "claude": 0.9  // 只配置了父级倍率
  }
}

GroupRatio = {
  "claude": 1.2,
  "claude-按量": 1.5
}

// 计算过程
userGroup := "vip"
usingGroup := "claude-按量"

// 1. 查找模型倍率
modelRatio, found := GetGroupGroupRatio("vip", "claude-按量")
// 先查 GroupGroupRatio["vip"]["claude-按量"] → 未找到
// 回退查 GroupGroupRatio["vip"]["claude"] → 找到 0.9
modelRatio = 0.9

// 2. 查找分组倍率
groupRatio := GetGroupRatio("claude-按量")
// 先查 GroupRatio["claude-按量"] → 找到 1.5
groupRatio = 1.5

// 3. 最终倍率 = 0.9 * 1.5 = 1.35

7. 配置说明

7.1 GroupRatio 配置

配置位置：系统设置 → 运营设置 → 分组倍率

配置格式：JSON 对象

配置示例：

{
  "default": 1.0,
  "vip": 0.9,
  "svip": 0.8,
  "claude": 1.2,
  "claude-包月": 1.0,
  "claude-按量": 1.5,
  "openai": 1.1,
  "openai-包月": 0.95,
  "openai-按量": 1.3
}

💡 倍率配置策略

  • 父级倍率：作为子级的默认倍率（子级未配置时使用）
  • 子级倍率：覆盖父级倍率（优先使用）
  • 独立分组：直接配置倍率，无继承关系

7.2 GroupGroupRatio 配置

配置位置：系统设置 → 运营设置 → 分组模型倍率

配置格式：JSON 嵌套对象

配置示例：

{
  "vip": {
    "claude": 0.9,           // 父级配置，作为所有 claude 子级的默认倍率
    "claude-按量": 0.85,      // 子级配置，覆盖父级
    "openai-包月": 0.8
  },
  "svip": {
    "claude": 0.8,
    "openai": 0.75
  }
}

7.3 GroupTree 配置

配置位置：系统设置 → 运营设置 → 分组层级树

配置格式：JSON 对象（父级→子级数组）

配置示例：

{
  "claude": ["claude-包月", "claude-按量"],
  "openai": ["openai-包月", "openai-按量"]
}

⚠️ GroupTree 配置限制

  • 每个子级只能属于一个父级（不能重复）
  • 不支持嵌套层级（子级不能同时是父级）
  • 父级和子级不能同名
  • 配置空对象 {} 会清空 GroupTree

7.4 配置建议

推荐配置方案

  1. 先规划分组结构：确定哪些分组应该成为父级，哪些是子级
  2. 配置 GroupTree：定义父子级关系
  3. 配置父级倍率：为父级分组配置基础倍率（作为子级默认值）
  4. 配置子级倍率：为需要差异化的子级单独配置倍率
  5. 配置模型倍率：优先配置父级的模型倍率，减少配置量

❌ 低效配置（全部配置子级）

GroupGroupRatio = {
  "vip": {
    "claude-包月": 0.9,
    "claude-按量": 0.9,
    "openai-包月": 0.85,
    "openai-按量": 0.85
  }
}

配置冗余，维护困难

✅ 高效配置（利用继承）

GroupGroupRatio = {
  "vip": {
    "claude": 0.9,
    "openai": 0.85
  }
}

配置简洁，子级自动继承

8. 使用场景

场景 1：多计费模式管理

需求

Claude 和 OpenAI 都有包月和按量两种计费模式，需要分别管理和计费

解决方案

// 1. 配置 GroupTree
{
  "claude": ["claude-包月", "claude-按量"],
  "openai": ["openai-包月", "openai-按量"]
}

// 2. 配置渠道
- 包月渠道：分组设置为 "claude-包月" 或 "openai-包月"
- 按量渠道：分组设置为 "claude-按量" 或 "openai-按量"

// 3. 创建套餐
- 按量套餐：channel_groups = ["claude-按量", "openai-按量"]
- 包月套餐：channel_groups = ["claude-包月", "openai-包月"]

// 4. 创建令牌
- 用户选择 "claude" 分组
- 自动授权 "claude-包月" 和 "claude-按量" 两个子级

优势

  • 用户只需选择 "claude"，无需关心计费模式
  • 系统根据套餐自动路由到正确的计费模式
  • 管理员可以为不同计费模式配置不同倍率

场景 2：差异化定价

需求

VIP 用户使用 Claude 享受 9 折，按量使用享受 8.5 折

解决方案

// 配置 GroupGroupRatio
{
  "vip": {
    "claude": 0.9,          // 父级默认 9 折
    "claude-按量": 0.85     // 按量特殊优惠 8.5 折
  }
}

// 计费逻辑
- VIP 用户使用 claude-包月：查找 "claude-按量" → 未找到 → 回退 "claude" → 0.9
- VIP 用户使用 claude-按量：查找 "claude-按量" → 找到 → 0.85

优势

  • 利用倍率回退机制，减少配置量
  • 可以为特定子级设置差异化定价
  • 配置清晰，易于理解和维护

场景 3：渠道分组简化

需求

新增 Claude 渠道，希望同时支持包月和按量用户

解决方案

❌ 不使用 GroupTree

需要手动填写：

claude-包月,claude-按量

容易遗漏或拼写错误

✅ 使用 GroupTree

只需填写：

claude

自动展开为所有子级，不会遗漏

场景 4：套餐故障切换

需求

用户当前套餐的 Claude 渠道全部故障，自动切换到备用套餐

解决方案

// 用户配置
- 主套餐：channel_groups = ["claude-按量"]
- 备用套餐：channel_groups = ["claude"]  // 父级分组
- 令牌分组：claude

// 故障切换流程
1. 主套餐 "claude-按量" 渠道全部故障
2. 检测到 auto_switch = 1，触发故障切换
3. 获取备用套餐，其 channel_groups = ["claude"]
4. 展开 "claude" → ["claude-包月", "claude-按量"]
5. 计算交集：令牌 "claude" 的子级 ∩ 套餐分组 = ["claude-包月", "claude-按量"]
6. 在 "claude-包月" 中找到可用渠道，切换成功

优势

  • 备用套餐配置父级分组，自动覆盖所有子级
  • 交集逻辑支持父级展开，匹配更灵活
  • 提高故障切换成功率

场景 5：渐进式迁移

需求

现有系统已经使用 "claude-包月" 和 "claude-按量"，希望引入父级分组简化管理

解决方案

  1. 第一步：配置 GroupTree 
    {
  "claude": ["claude-包月", "claude-按量"]
}
  2. 第二步：系统自动生效
    • 现有渠道、套餐、令牌保持不变
    • 渠道缓存自动刷新（父级展开）
    • 新令牌创建时只显示 "claude"
  3. 第三步：逐步迁移
    • 新套餐使用 "claude" 父级分组
    • 新渠道使用 "claude" 父级分组
    • 旧配置自然过渡，无需手动修改

优势

  • 无需数据迁移，配置即生效
  • 新旧配置并存，平滑过渡
  • 逐步替换，风险可控
