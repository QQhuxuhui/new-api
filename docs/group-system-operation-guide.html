🎯 分组系统运营配置指南

非技术人员也能轻松掌握的配置流程

适用角色：平台运营、业务管理员

📖 5分钟快速理解

什么是分组系统？

分组系统就像商品分类，帮您：

  • 管理渠道：把不同的 AI 服务商（如 Claude、OpenAI）分门别类
  • 控制权限：决定哪些用户可以用哪些服务
  • 差异定价：对不同用户群体设置不同价格
🌰 举个例子

场景：您的平台同时提供 Claude 和 OpenAI 服务，每种都有包月和按量计费两种模式。

❌ 不用层级分组（复杂）

创建令牌时，用户看到 6 个选项：

  • claude-包月
  • claude-按量
  • openai-包月
  • openai-按量
  • gemini
  • default

用户容易选错，配置繁琐

✅ 使用层级分组（简洁）

创建令牌时，用户只看到 4 个选项：

  • claude（自动包含包月和按量）
  • openai（自动包含包月和按量）
  • gemini
  • default

用户选择简单，系统自动处理

✨ 核心概念（只需记住这三个）

名称 说明 例子
父级分组 大类别，包含多个子级 claude、openai
子级分组 具体分类，隶属于父级 claude-包月、claude-按量
独立分组 既不是父级也不是子级 default、gemini

不确定怎么规划？先看下面的"常见配置场景"

2 配置分组层级树（GroupTree）

您需要做什么：

  • 登录管理后台

  • 填写 JSON 配置（下面有模板）

这一步之后，系统会自动刷新缓存，立即生效

3 配置分组倍率（可选）

您需要做什么：

  • 进入：系统设置 → 运营设置 → 分组倍率
  • 为父级分组设置基础倍率（子级会继承）
  • 为需要特殊定价的子级单独设置倍率
倍率 = 最终价格系数，1.0 表示原价，0.9 表示 9 折，1.5 表示 1.5 倍价格

新建渠道时，选择父级分组更方便，不会遗漏子级

5 配置套餐分组（如使用套餐系统）

您需要做什么：

  • 进入：套餐管理 → 编辑套餐

  • 可以选择父级（覆盖所有子级）或子级（精确控制）

您需要做什么：

  • 创建测试令牌，检查分组选项是否正确

  • 检查计费倍率是否正确

✅ 验证通过标准

  • 令牌创建时只显示父级分组和独立分组
  • API 调用能正常路由到渠道
  • 费用计算符合倍率设置

📝 详细操作步骤

步骤 1：规划分组结构

如何规划？

问自己三个问题：

  1. 平台提供哪些 AI 服务？（claude、openai、gemini...）
  2. 每种服务有哪些计费模式？（包月、按量...）
  3. 用户需要分别管理这些模式吗？

决策树：

  • 需要细分管理 → 设置为父子级关系
  • 不需要细分 → 设置为独立分组

示例规划表：

服务 是否细分 父级分组 子级分组
Claude 是（包月/按量） claude claude-包月, claude-按量
OpenAI 是（包月/按量） openai openai-包月, openai-按量
Gemini 否 - gemini（独立分组）
默认 否 - default（独立分组）

步骤 2：配置 GroupTree

操作指南

① 打开配置页面

登录管理后台 → 系统设置 → 运营设置 → 找到"分组层级树（GroupTree）"

② 填写配置（JSON 格式）

根据您的规划表，填写配置：

{ // 父级分组名: [子级分组1, 子级分组2, ...] "claude": ["claude-包月", "claude-按量"], "openai": ["openai-包月", "openai-按量"] }
注意事项：
  • 父级和子级的名称要与您已有的分组名称一致
  • 独立分组（如 gemini、default）不要写在这里
  • 确保 JSON 格式正确（大括号、逗号、引号）

③ 保存并验证

点击"保存"按钮，系统会自动验证配置：

  • ✅ 保存成功 → 配置立即生效
  • ❌ 保存失败 → 根据错误提示修改

步骤 3：配置分组倍率

定价策略配置

① 打开配置页面

系统设置 → 运营设置 → 分组倍率（GroupRatio）

② 配置倍率（推荐策略）

{ // 独立分组 "default": 1.0, // 默认分组原价 "gemini": 0.95, // Gemini 9.5 折 // 父级分组（作为子级的默认倍率） "claude": 1.2, // Claude 默认 1.2 倍 "openai": 1.1, // OpenAI 默认 1.1 倍 // 子级分组（覆盖父级倍率） "claude-包月": 1.0, // 包月优惠到原价 "claude-按量": 1.5, // 按量 1.5 倍 "openai-按量": 1.3 // OpenAI 按量 1.3 倍 }
倍率继承机制：
子级未配置倍率时，会自动使用父级倍率。
例如：openai-包月 未配置，会使用 openai 的 1.1 倍率

③ 实际计费示例

用户使用 查找倍率 最终倍率
claude-包月 找到子级配置 1.0（原价）
claude-按量 找到子级配置 1.5
openai-包月 子级未配置 → 使用父级 1.1
gemini 独立分组配置 0.95

步骤 4：配置渠道

渠道分组设置

① 新建或编辑渠道

渠道管理 → 新建渠道（或编辑现有渠道）

② 选择分组

传统方式

手动输入所有分组：

claude-包月,claude-按量

❌ 容易遗漏或拼写错误

推荐方式（使用父级）

只选择父级分组：

claude

✅ 自动展开到所有子级

③ 适用场景

场景 配置方式 说明
渠道支持所有 Claude 用户 选择 claude 包月和按量用户都能用
渠道仅支持包月用户 选择 claude-包月 只有包月用户能用
渠道支持多种服务 选择 claude,openai 两个父级都会展开

步骤 5：配置套餐（如使用）

套餐分组配置

① 打开套餐配置

套餐管理 → 新建套餐（或编辑现有套餐）

② 设置渠道分组

配置策略：

  • 包月套餐：只选择包月分组（如 claude-包月, openai-包月）
  • 按量套餐：只选择按量分组（如 claude-按量, openai-按量）
  • 通用套餐：选择父级分组（如 claude, openai）
套餐分组决定了这个套餐可以访问哪些渠道，建议根据业务模式精确配置

③ 配置示例

套餐名称 渠道分组配置 用户权限
免费试用 default 只能用默认渠道
基础包月 claude-包月, openai-包月 包月渠道
按量付费 claude-按量, openai-按量, gemini 按量渠道 + gemini
企业套餐 claude, openai, gemini 所有渠道（父级自动展开）

步骤 6：测试验证

上线前检查清单

应该只显示父级分组和独立分组，不显示子级分组

应该能正常路由到该父级下的所有子级渠道

倍率应该符合配置（查看消费记录）

检查 auto_switch 是否正常工作

确认没有错误日志，分组展开正常

🎬 常见配置场景

场景 1：多计费模式管理

业务需求：

平台提供包月和按量两种计费模式，需要分别管理定价和渠道

配置方案：

  1. GroupTree 配置：
    { "claude": ["claude-包月", "claude-按量"], "openai": ["openai-包月", "openai-按量"] }
  2. 分组倍率配置：
    { "claude-包月": 1.0, // 包月优惠价 "claude-按量": 1.5, // 按量正常价 "openai-包月": 0.95, // 包月 95 折 "openai-按量": 1.3 // 按量 1.3 倍 }
  3. 创建套餐：
    • 包月套餐：选择 claude-包月, openai-包月
    • 按量套餐：选择 claude-按量, openai-按量
  4. 用户创建令牌：

    用户只需选择 claude 或 openai，系统根据套餐自动路由到对应计费模式

✅ 效果

  • 用户界面简洁，不需要理解计费模式
  • 系统自动按套餐路由到正确的渠道
  • 差异化定价灵活实现
场景 2：会员等级定价

业务需求：

VIP 会员使用 Claude 享 9 折，按量使用享 8.5 折

配置方案：

  1. GroupTree 配置：（同场景 1）
  2. 分组模型倍率配置（GroupGroupRatio）：

    系统设置 → 运营设置 → 分组模型倍率

    { "vip": { "claude": 0.9, // VIP 使用 Claude 默认 9 折 "claude-按量": 0.85 // VIP 使用按量特惠 8.5 折 } }
  3. 倍率计算示例：
    用户等级 使用分组 倍率计算 最终倍率
    VIP claude-包月 查 claude-按量 未找到 → 查 claude → 0.9 0.9（9折）
    VIP claude-按量 查 claude-按量 → 找到 0.85（8.5折）
    普通用户 claude-按量 无会员倍率，使用分组倍率 1.5（正常价）

✅ 效果

  • 利用倍率继承，只需配置父级和特殊子级
  • VIP 会员自动享受优惠
  • 配置简洁，易于维护
场景 3：渐进式升级

业务需求：

现有系统已经在用，不想一次性大改，希望平滑过渡

配置方案：

  1. 第 1 天：配置 GroupTree
    • 只配置 GroupTree，不改其他
    • 系统自动刷新缓存
    • 现有配置继续工作
  2. 第 2-7 天：观察期
    • 监控日志，确认无异常
    • 渠道路由正常
    • 计费准确
  3. 第 8 天起：逐步迁移
    • 新建渠道时使用父级分组
    • 新建套餐时使用父级分组
    • 引导新用户使用父级分组
    • 旧配置保持不变，自然过渡

💡 关键优势

  • 无需数据迁移
  • 新旧配置并存
  • 风险可控
  • 可随时回滚（删除 GroupTree 配置）

❓ 常见问题

Q1: 配置 GroupTree 后，现有的令牌会受影响吗？

A: 不会。现有令牌的分组配置保持不变，继续正常工作。只有新创建的令牌会看到新的分组选项。

Q2: 我能删除 GroupTree 配置吗？

A: 可以。将配置改为空对象 {} 保存即可。系统会恢复到原来的扁平分组模式。

Q3: 子级分组可以属于多个父级吗？

A: 不可以。每个子级只能属于一个父级，否则保存时会报错。

Q4: 配置错误了怎么办？

A: 系统会在保存时自动校验，如果有错误会提示。常见错误：

  • JSON 格式错误（少逗号、引号不匹配）
  • 子级重复（一个子级出现在多个父级中）
  • 嵌套层级（子级同时也是父级）

Q5: 倍率配置很复杂，有简化方法吗？

A: 有！利用倍率继承机制：

  • 只为父级配置统一倍率
  • 只为需要差异化的子级单独配置
  • 其他子级自动继承父级倍率

Q6: 如何知道配置生效了？

A: 三个检查点：

  1. 保存后系统显示"保存成功"
  2. 创建令牌时，分组选项已更新
  3. 查看系统日志，有"GroupTree updated"记录
