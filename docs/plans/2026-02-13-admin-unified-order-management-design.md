# 管理员订单页面合并充值订单

## 目标
将钱包充值订单（topup_orders）合并到管理员的套餐订单页面，统一查询和管理。

## 方案
后端应用层合并查询（方案 A），与用户端 MyOrders 一致的模式。

## 后端改动
- `GetAllPlanOrders` 新增 `order_type` 参数，同时查两张表合并排序分页
- 新增充值订单管理 API：手动完成、取消、删除、详情
- 路由注册在 adminRoute 下

## 前端改动
- AdminOrders 新增类型筛选（全部/套餐/充值）
- 表格列适配：类型 Tag、名称列、操作按钮根据类型调不同 API
- 详情弹窗根据订单类型展示不同字段
- 侧边栏菜单名更新为"订单管理"
