# 统一订单管理页面设计

## 概述

将管理员后台的套餐订单和充值订单合并到统一的"订单管理"页面，方便管理员在一个地方查看所有订单类型，并显示用户信息。

## 需求背景

- 当前套餐订单和充值订单分散在不同页面
- 充值订单（钱包账单）页面缺少用户信息展示
- 管理员需要在多个页面切换查看不同类型订单

## 设计方案

### 1. 整体架构

- 将现有 `AdminOrders` 页面升级为统一的"订单管理"页面
- 通过 Tab 切换显示套餐订单和充值订单
- 保留原有钱包管理中的账单功能不变

### 2. 后端 API

#### 新增管理员充值订单 API

**端点：** `GET /api/user/topup-orders`（管理员权限）

**请求参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| page | int | 页码，默认 1 |
| page_size | int | 每页数量，默认 10 |
| status | string | 订单状态 (pending/paid/expired/cancelled) |
| user_id | int | 按用户 ID 筛选 |
| username | string | 按用户名模糊搜索 |
| email | string | 按邮箱模糊搜索 |
| payment_method | string | 按支付方式筛选 |
| min_amount | float | 最小金额 |
| max_amount | float | 最大金额 |
| start_time | int64 | 开始时间戳 |
| end_time | int64 | 结束时间戳 |
| order_no | string | 订单号搜索 |

**响应数据：**

```json
{
  "success": true,
  "data": {
    "orders": [
      {
        "id": 1,
        "order_no": "TOP20240128...",
        "user_id": 100,
        "username": "testuser",
        "email": "test@example.com",
        "amount": 10.00,
        "quota": 500000,
        "final_price": 9.50,
        "discount_rate": 0.95,
        "payment_method": "epay",
        "status": "paid",
        "created_at": 1706428800,
        "paid_at": 1706429000
      }
    ],
    "total": 100,
    "page": 1,
    "page_size": 10
  }
}
```

### 3. 前端页面设计

#### 页面布局

```
┌─────────────────────────────────────────────────────────────┐
│  订单管理                                                    │
├─────────────────────────────────────────────────────────────┤
│  [套餐订单] [充值订单]  ← Tab 切换                            │
├─────────────────────────────────────────────────────────────┤
│  筛选区域：                                                  │
│  订单状态 | 支付方式 | 时间范围 | 金额范围                    │
│  搜索：订单号/用户名/用户ID/邮箱                              │
├─────────────────────────────────────────────────────────────┤
│  订单列表表格（可展开行）                                     │
└─────────────────────────────────────────────────────────────┘
```

#### 表格设计

**收起状态列：**
- 展开按钮
- 订单号
- 用户名
- 金额
- 支付方式
- 状态
- 创建时间
- 操作

**展开区域内容：**
- 用户信息：用户ID、邮箱
- 订单详情：
  - 充值订单：充值金额、获得额度、折扣率、支付单号、支付时间
  - 套餐订单：套餐名称、额度、有效期、交付状态、关联用户套餐ID

#### 操作按钮（根据状态显示）

| 状态 | 可用操作 |
|------|----------|
| pending | 取消 |
| paid（套餐订单，未交付） | 手动完成、取消 |
| delivered（已交付） | 删除 |
| expired/cancelled | 删除 |

#### 状态标签颜色

- pending（待支付）：橙色
- paid（已支付）：蓝色
- delivered（已交付）：绿色
- expired（已过期）：灰色
- cancelled（已取消）：红色
