💰 钱包充值用户分组策略说明

不使用套餐系统的用户如何匹配渠道分组

📖 核心概念

什么是钱包充值用户？

钱包充值用户是指直接在账户余额充值，不购买套餐的用户。这类用户的特点是：

  • 按实际使用量扣除余额（按量付费）
  • 没有套餐限制，可以灵活使用
  • 通过令牌或用户分组来控制可访问的渠道

💼 套餐用户

计费方式：使用套餐额度

渠道控制：由套餐的"渠道分组"决定

优势：套餐定价、限流、配额管理

💰 钱包充值用户

计费方式：直接扣除账户余额

渠道控制：由令牌或用户分组决定

优势：灵活、简单、即充即用

🎯 分组匹配流程

步骤 1：用户使用令牌调用 API
系统接收到请求，开始分组匹配
↓
步骤 2：检查令牌是否配置了分组
查看令牌的"分组"字段
↓
✅ 情况 A：令牌配置了分组（如 "claude"）
使用令牌的分组 → 跳转到步骤 4
❌ 情况 B：令牌未配置分组
使用用户的默认分组 → 继续到步骤 3
↓
步骤 3：获取用户的默认分组
从用户表中读取用户的 Group 字段
↓
步骤 4：展开父级分组（如使用 GroupTree）
如果分组是父级（如 "claude"），自动展开为子级（如 "claude-包月", "claude-按量"）
↓
步骤 5：在分组中查找可用渠道
从渠道缓存中随机选择健康的渠道
↓
步骤 6：路由到渠道，开始计费
使用分组倍率计算费用，从用户余额扣除

🔑 关键结论

钱包充值用户的分组优先级：

  1. 优先级 1 令牌的分组配置（如果令牌设置了分组）
  2. 优先级 2 用户的默认分组（如果令牌没有设置分组）

✨ 父级分组支持：

令牌和用户可以使用父级分组（如 claude），系统会自动展开为所有子级分组（如 claude-包月, claude-按量），并在这些子级中选择可用渠道。

⚙️ 配置策略

策略 1：使用令牌分组（推荐）

适用场景

希望不同令牌访问不同的服务，精细化控制权限

配置步骤：

  1. 配置 GroupTree（如需要）：
    { "claude": ["claude-包月", "claude-按量"], "openai": ["openai-包月", "openai-按量"] }
  2. 创建令牌时选择分组：
    • 令牌 A：分组选择 claude
    • 令牌 B：分组选择 openai
    • 令牌 C：分组选择 default
  3. 用户使用不同令牌：
    令牌 分组配置 实际展开 可访问渠道
    令牌 A claude claude-包月, claude-按量 所有 Claude 渠道
    令牌 B openai openai-包月, openai-按量 所有 OpenAI 渠道
    令牌 C default default（独立分组） 默认渠道

✅ 优势

  • 灵活控制：不同令牌访问不同服务
  • 精细管理：可以为不同应用/项目创建专用令牌
  • 安全隔离：一个令牌泄露不影响其他服务

策略 2：使用用户默认分组

适用场景

用户的所有令牌使用相同的服务范围，简化管理

配置步骤：

  1. 设置用户的默认分组：

    用户管理 → 编辑用户 → 设置"分组"字段

    • VIP 用户：设置为 vip
    • 普通用户：设置为 default
    • 企业用户：设置为 enterprise
  2. 创建令牌时不选择分组：

    令牌的"分组"字段留空或选择 "auto"

  3. 用户使用令牌：
    用户等级 用户默认分组 令牌分组 实际使用分组
    VIP 用户 vip （留空） vip
    普通用户 default （留空） default
    企业用户 enterprise （留空） enterprise

✅ 优势

  • 统一管理：用户等级决定服务范围
  • 简化配置：不需要为每个令牌单独设置分组
  • 自动升级：修改用户分组，所有令牌自动生效

策略 3：混合策略（灵活）

适用场景

大部分令牌使用用户默认分组，特殊令牌单独配置

示例：企业用户的多令牌管理

用户默认分组：enterprise（可访问所有服务）

令牌用途 令牌分组配置 实际使用分组 说明
主令牌 （留空） enterprise 使用用户默认分组
测试令牌 default default 限制在测试环境
Claude 专用令牌 claude claude 只能访问 Claude 服务
OpenAI 专用令牌 openai openai 只能访问 OpenAI 服务

优势：默认使用企业分组（权限最大），特殊用途的令牌单独限制，实现精细化管理。

🌳 结合 GroupTree 层级分组

GroupTree 对钱包充值用户的影响

当配置了 GroupTree 后，父级分组会自动展开为子级分组，让钱包充值用户也能享受层级分组的便利。

示例场景：多计费模式

业务需求

平台提供 Claude 服务，有包月渠道和按量渠道两种

配置方案

  1. 配置 GroupTree：
    { "claude": ["claude-包月", "claude-按量"] }
  2. 配置渠道：
    • 包月渠道：分组设置为 claude-包月
    • 按量渠道：分组设置为 claude-按量
  3. 创建令牌：
    • 用户创建令牌时，分组选择 claude（父级分组）
    • 系统会自动展开为 ["claude-包月", "claude-按量"]
  4. 路由逻辑：
    • 用户使用令牌调用 API
    • 系统从 claude-包月 和 claude-按量 中随机选择可用渠道
    • 可能路由到包月渠道，也可能路由到按量渠道

✅ 效果

  • 用户只需选择 "claude"，无需关心计费模式
  • 系统自动在包月和按量渠道间负载均衡
  • 渠道管理员可以灵活调整计费模式的渠道分配

差异化定价

业务需求

包月渠道价格便宜（1.0 倍率），按量渠道价格贵（1.5 倍率）

配置方案

  1. 配置分组倍率：
    { "claude-包月": 1.0, "claude-按量": 1.5 }
  2. 计费逻辑：
    用户使用 实际路由到 分组倍率 最终费用
    claude 令牌 claude-包月 渠道 1.0 基础价格 × 1.0
    claude 令牌 claude-按量 渠道 1.5 基础价格 × 1.5

⚠️ 注意

使用父级分组（如 claude）时，系统会随机选择包月或按量渠道，用户的费用可能有波动。

建议：如果希望用户固定使用某种计费模式，应该在令牌中直接配置子级分组（如 claude-包月 或 claude-按量）。

❓ 常见问题

Q1: 钱包充值用户能使用套餐的渠道吗？

A: 可以。只要渠道的分组与用户/令牌的分组匹配，就能访问。套餐系统和钱包充值系统是共享渠道资源的。

Q2: 如果令牌分组和用户分组不一样，以哪个为准？

A: 令牌分组优先。如果令牌配置了分组，就使用令牌的分组；否则才使用用户的默认分组。

Q3: 用户创建令牌时，能看到哪些分组？

A: 取决于用户的权限配置：

  • 如果配置了 GroupTree，只显示父级分组和独立分组
  • 用户能看到的分组 = 用户等级允许的分组 ∩ 系统配置的分组

Q4: 如何限制钱包充值用户只能用特定渠道？

A: 有两种方法：

  1. 方法 1：修改用户的默认分组，并在令牌中不配置分组
  2. 方法 2：在令牌中明确配置分组，覆盖用户的默认分组

Q5: 钱包充值用户的计费倍率如何计算？

A: 计费倍率 = 用户等级倍率（GroupGroupRatio）× 分组倍率（GroupRatio）× 渠道倍率

如果使用父级分组，子级倍率会继承父级倍率（如果子级没有单独配置）。

Q6: 套餐用户和钱包充值用户能同时存在吗？

A: 可以。同一个用户可以既有套餐，又有余额：

  • 当用户有可用套餐时，优先使用套餐额度
  • 当套餐额度用完或无可用套餐时，使用余额扣费

💡 配置建议

推荐配置方案

用户类型 推荐策略 原因
个人用户 令牌分组策略 灵活，可以为不同项目创建不同令牌
企业用户 混合策略 主令牌使用默认分组，特殊令牌单独配置
API 服务商 用户默认分组 按用户等级统一管理权限
开发/测试团队 令牌分组策略 隔离生产环境和测试环境

配置 GroupTree 的建议

  • 对于钱包充值用户：建议配置 GroupTree，简化令牌创建
  • 父级分组命名：使用服务商名称（如 claude、openai）
  • 子级分组命名：加上计费模式后缀（如 claude-包月、claude-按量）
  • 倍率配置：优先配置父级倍率，子级只配置差异

需要注意的陷阱

  • 陷阱 1：令牌配置了父级分组，用户费用可能在不同子级间波动
  • 陷阱 2：修改用户默认分组会影响该用户所有未配置分组的令牌
  • 陷阱 3：删除分组前，确保没有用户/令牌正在使用
