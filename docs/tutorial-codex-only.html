OpenAI Codex CLI 使用教程

跟着这个教程，轻松在你的电脑上安装并使用 Codex CLI

0

前置要求

支持平台：

  • macOS 10.15+
  • Linux（Ubuntu、Debian、Fedora 等）
  • Windows（仅通过 WSL 支持）

环境依赖：

node --version
💡 提示： Codex CLI 依赖 Node.js 环境，请确保已安装最新 LTS 版本。
1

安装 Node.js 环境

Codex CLI 需要 Node.js 环境才能运行。请根据你的操作系统选择安装方式：

🪟 Windows

Codex CLI 在 Windows 上仅通过 WSL（Windows Subsystem for Linux）支持。请先安装 WSL，然后在 WSL 环境中按照 Linux 步骤操作。

安装 WSL：

# 在 PowerShell（管理员）中运行
wsl --install

安装完成后重启电脑，然后在 WSL 终端中继续操作。

在 WSL 中安装 Node.js：

# Ubuntu/Debian (WSL 默认)
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
🍎 macOS

方法一：使用 Homebrew（推荐）

# 更新 Homebrew
brew update

# 安装 Node.js
brew install node

方法二：官网下载

  1. 访问 nodejs.org
  2. 下载适合 macOS 的 LTS 版本
  3. 打开下载的 .pkg 文件，按照安装程序指引完成安装
💡 macOS 注意事项：
  • 如果遇到权限问题，可能需要使用 sudo
  • 首次运行可能需要在系统偏好设置中允许
  • 建议使用 Terminal 或 iTerm2
🐧 Linux

方法一：使用 NodeSource 仓库（推荐）

# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs

方法二：使用系统包管理器

# Ubuntu/Debian
sudo apt update
sudo apt install nodejs npm

# CentOS/RHEL/Fedora
sudo dnf install nodejs npm
💡 Linux 注意事项：
  • 某些发行版可能需要安装额外的依赖
  • 如果遇到权限问题，使用 sudo
  • 确保你的用户在 npm 的全局目录有写权限

验证安装：

node --version
npm --version

如果显示版本号，说明安装成功了！

2

安装 Codex CLI

根据平台选择安装方式：

🪟 Windows (WSL)

在 WSL 终端中使用 npm 安装：

npm install -g @openai/codex

如果遇到权限问题，请尝试 sudo npm install -g @openai/codex

⚠️ 注意： Codex CLI 不直接支持 Windows，请确保在 WSL 环境中操作。
🍎 macOS（Homebrew 推荐）
brew install --cask codex

也可以使用 npm：npm install -g @openai/codex

🐧 Linux（npm）
npm install -g @openai/codex

如果遇到权限问题，请尝试 sudo npm install -g @openai/codex

验证安装：

codex --version

如果显示版本号，说明 Codex CLI 已经成功安装了。

3

配置中转 API

Codex CLI 的配置分为两部分：配置文件和环境变量。

1. 编辑配置文件

编辑 ~/.codex/config.toml，在文件开头添加以下配置：

model_provider = "crs"
model = "gpt-5-codex"
model_reasoning_effort = "high"
disable_response_storage = true
preferred_auth_method = "apikey"

[model_providers.crs]
name = "crs"
base_url = "{{OPENAI_API_URL}}"
wire_api = "responses"
requires_openai_auth = true
env_key = "CRS_OAI_KEY"

2. 编辑认证文件

编辑 ~/.codex/auth.json，将 OPENAI_API_KEY 设置为 null：

{
  "OPENAI_API_KEY": null
}
📝 说明： 将 OPENAI_API_KEY 设置为 null，是为了让 Codex 使用我们在 config.toml 中配置的 env_key（即 CRS_OAI_KEY）来读取 API 密钥。

3. 设置环境变量 CRS_OAI_KEY

请根据你的操作系统设置环境变量：

🪟 Windows (WSL)

临时设置（当前会话）：

export CRS_OAI_KEY="你的API密钥"

永久设置：

# WSL 中默认使用 bash
echo 'export CRS_OAI_KEY="你的API密钥"' >> ~/.bashrc
source ~/.bashrc

💡 WSL 环境中的操作与 Linux 相同。

🍎 macOS

临时设置（当前会话）：

export CRS_OAI_KEY="你的API密钥"

永久设置：

# 对于 zsh（macOS 默认）
echo 'export CRS_OAI_KEY="你的API密钥"' >> ~/.zshrc
source ~/.zshrc

# 对于 bash
echo 'export CRS_OAI_KEY="你的API密钥"' >> ~/.bash_profile
source ~/.bash_profile

💡 验证：echo $CRS_OAI_KEY

🐧 Linux

临时设置（当前会话）：

export CRS_OAI_KEY="你的API密钥"

永久设置：

# 对于 bash（Linux 默认）
echo 'export CRS_OAI_KEY="你的API密钥"' >> ~/.bashrc
source ~/.bashrc

# 对于 zsh
echo 'export CRS_OAI_KEY="你的API密钥"' >> ~/.zshrc
source ~/.zshrc

💡 验证：echo $CRS_OAI_KEY

📝 配置要点：
  • API Base URL：{{OPENAI_API_URL}}
  • API Key：将 你的API密钥 替换为你的实际密钥（格式如 cr_xxxxxxxxxx）
  • 环境变量名：CRS_OAI_KEY，与 config.toml 中的 env_key 对应
4

开始使用 Codex CLI

基本命令模式：

codex "list all files in current directory"

Codex 会将你的自然语言描述转换为对应的 Shell 命令。

在特定项目中使用：

# 进入你的项目目录
cd /path/to/your/project

# 使用 Codex
codex "explain the main function in this project"

常用功能：

codex "描述任务" 自然语言转命令
codex --explain 解释命令含义
codex --version 查看版本
?

常见问题解决

安装时提示 "permission denied" 错误

  • macOS/Linux/WSL：使用 sudo npm install -g @openai/codex，或配置 npm 使用用户目录：npm config set prefix ~/.npm-global 并添加到 PATH

Windows 无法直接运行 Codex

Codex CLI 仅支持 macOS 和 Linux。Windows 用户请先安装 WSL：

wsl --install

安装完成后在 WSL 终端中操作即可。

环境变量设置后不生效

  • 确认修改了正确的文件（.bashrc 或 .zshrc）
  • 执行 source 命令或重启终端
  • 验证设置：echo $CRS_OAI_KEY

配置文件不存在

首次使用时 ~/.codex/ 目录可能不存在，手动创建即可：

mkdir -p ~/.codex
touch ~/.codex/config.toml
touch ~/.codex/auth.json
