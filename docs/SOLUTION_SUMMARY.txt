📋 问题：高峰期用户调用模型A​PI时，首字节响应时间超过30秒

🎯 根本原因：HTTP客户端连接池配置不足
   - Go默认每个主机只保持2个空闲连接
   - 高峰期并发请求超过2个时，需要重新建立TCP+TLS连接
   - 到海外A​PI建立连接需要5-10秒，导致请求排队

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 已完成的优化

1. 优化HTTP客户端连接池（service/http_client.go）
   ✓ MaxIdleConnsPerHost: 2 → 100
   ✓ MaxConnsPerHost: 0 → 200
   ✓ MaxIdleConns: 100 → 1000
   ✓ 连接超时: 30秒 → 10秒
   ✓ 响应头超时: 无 → 60秒

2. 优化Docker配置（docker-compose.yml）
   ✓ RELAY_TIMEOUT=120秒
   ✓ STREAMING_TIMEOUT=300秒
   ✓ DNS优化: 8.8.8.8, 1.1.1.1

💡 核心优化原理

问题：
  并发请求 > 2 → 需要新建连接 → TCP握手(1-2秒) + TLS握手(2-3秒)
  → 多个请求排队等待 → TTFB 30秒+

解决：
  增加连接池 → 复用现有连接 → 跳过握手过程 → TTFB 3-8秒
