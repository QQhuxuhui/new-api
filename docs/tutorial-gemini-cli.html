✨ Google Gemini CLI 安装与配置

前置要求

系统要求：

  • Node.js 18+ 或更高版本（推荐 Node 20+）
  • 支持 Windows、macOS 和 Linux

检查 Node.js 版本：

node --version

💡 如果版本低于 18 或未安装，请先完成步骤 1

步骤 1: 安装 Node.js（如果需要）

Windows：

访问 nodejs.org 下载 LTS 版本安装包，运行安装程序，按默认选项安装，安装完成后重启终端。

macOS：

使用 Homebrew 安装（推荐）：

brew install node

Linux：

使用 NodeSource 仓库安装：

# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证安装
node --version
npm --version

步骤 2: 安装 Gemini CLI

使用 npm 全局安装（所有平台通用）：

npm install -g @google/gemini-cli

💡 权限问题解决：
• Windows：使用管理员权限运行 PowerShell 或 CMD
• macOS/Linux：如遇权限问题，使用 sudo npm install -g @google/gemini-cli

验证安装：

gemini --version

显示版本号即表示安装成功

步骤 3: 配置 API Key

获取 Gemini API Key：

  1. 访问 Google AI Studio
  2. 登录您的 Google 账户
  3. 在控制台中创建新的 API Key

配置 API Key（推荐方式 - 使用环境变量）：

Windows：

在 PowerShell 中执行：

# 临时设置（当前会话）
$env:GEMINI_API_KEY="YOUR_API_KEY"

# 永久设置（添加到用户环境变量）
[System.Environment]::SetEnvironmentVariable('GEMINI_API_KEY', 'YOUR_API_KEY', 'User')

macOS / Linux：

编辑 shell 配置文件（~/.bashrc 或 ~/.zshrc）：

# 添加以下行
export GEMINI_API_KEY="YOUR_API_KEY"

# 重新加载配置
source ~/.bashrc  # 或 source ~/.zshrc

验证配置：

# 首次运行时会使用环境变量中的 API Key
gemini

📝 配置说明：

  • 推荐方式: 使用 GEMINI_API_KEY 环境变量
  • API Key: 将 YOUR_API_KEY 替换为您从 Google AI Studio 获取的实际 API Key
  • 配置文件: Gemini CLI 也支持通过配置文件扩展功能，但 API Key 推荐使用环境变量

使用中转 API（如需要）：

如果使用中转服务，可能需要额外配置。请参考中转服务商的具体说明。

步骤 4: 开始使用

基本使用：

# 启动交互式对话
gemini

# 直接提问
gemini "What is the capital of France?"

# 使用管道输入
echo "Explain quantum computing" | gemini

常用功能：

  • 代码生成和解释
  • 文本翻译和摘要
  • 问答和知识查询
  • 创意写作辅助

💡 Gemini CLI 支持多种内置工具，可用于文件处理、代码分析等任务
