# 邀请奖励服务条款 (草案)

>
> 对应 spec Requirement: `User Agreement Affiliate Clause`

## 第 X 章 邀请奖励规则

### 一、奖励形式

1. 您通过邀请链接邀请的好友(以下简称"被邀请人")在本平台完成**真实支付**(不含兑换码、管理员手工赠送、活动赠送等非支付来源)后,您可获得相应金额的**站内消费额度**(以下简称"返佣额度",字段名 `AffQuota`)作为奖励。

2. 返佣额度**仅限本平台站内消费使用**(模型 API、套餐购买等),**不可提现为现金**或其他形式的法定货币。

3. 当前默认返佣比例为 **{{InviterRewardDefaultPercent}}%**(平台保留调整权)。具体当前比例与冷却天数,以您账户中"邀请奖励"页面显示的实时数值为准。

### 二、到账规则

1. 被邀请人每笔成功支付后,您的对应返佣记录会进入"冷却期",冷却期默认为 **{{InviterRewardCooldownDays}} 天**。

2. 冷却期满且通过反作弊审核后,返佣额度自动到账您的账户。

3. 在冷却期内,如果该笔支付发生退款,系统将自动撤销对应的返佣记录,您不会获得该笔奖励。

### 三、反作弊规则

为保证奖励的公平性与平台秩序,系统会对以下行为进行识别并**拒绝发放奖励**:

1. **同一 IP**:邀请人与被邀请人在最近 24 小时内有重叠的登录 IP;
2. **同一支付账号**:邀请人与被邀请人使用了相同的第三方支付账号(同一支付宝、微信、Stripe 客户等);
3. **邀请人分销资格已被冻结**(见下条)。

被识别为以上情况的返佣会被标记为"已拒绝"且不再到账。该状态为终态,请避免任何形式的自邀刷奖行为。

### 四、分销资格冻结

如发现以下情况,平台保留**冻结您的分销资格**(此后您邀请的新支付不再产生任何返佣)的权利:

- 多次或大规模触发反作弊规则;
- 通过技术手段或社会工程绕过反作弊机制;
- 在第三方平台进行虚假宣传、误导他人注册;
- 其他违反平台用户协议的行为。

### 五、平台保留的权利

1. 平台可随时调整返佣比例与冷却天数,新规则仅适用于调整后产生的新返佣记录,**不影响**已经写入"冷却中"或"已到账"的奖励金额。

2. 平台可随时通过 `EnableAffAutoSettle = false` 暂停整个分销奖励项目;暂停期间产生的返佣记录会保留,等项目恢复后继续按规则结算。

3. 对于头部分销者,平台可与其在线下沟通采用其他形式的奖励发放(如线下转账等)。线下已发放的奖励对应的 audit log 会被管理员标记为"线下已返现"且不再走站内额度结算,以避免重复发放。

### 六、隐私保护

1. 您**只能**在邀请奖励页看到聚合数字(已邀请人数、可用额度、累计额度、冷却中金额、本月新增、当前比例、冷却天数、分销状态);
2. 您**不能**看到任何具体的下级用户身份、订单号、支付方式或支付时间;
3. 平台管理员保留完整的审计访问权限以支持反作弊与对账,但内部访问受平台权限管理与日志记录约束。

### 七、注册与同意

1. 在本平台注册并使用邀请链接的行为,即视为您已阅读并同意本章规则。
2. 如您对本规则有疑问,请在使用邀请功能前联系平台客服。
