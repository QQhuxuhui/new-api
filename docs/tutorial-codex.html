🤖 OpenAI Codex CLI 安装与配置

系统要求

  • 支持 macOS 或 Linux（Windows 需使用 WSL）
  • Node.js（如使用 npm 安装）

检查 Node.js 版本：

node --version

💡 如果未安装 Node.js，请先完成步骤 1

步骤 1: 安装 Node.js（如果需要）

Windows：

访问 nodejs.org 下载 LTS 版本安装包，运行安装程序，按默认选项安装，安装完成后重启终端。

macOS：

使用 Homebrew 安装（推荐）：

brew install node

Linux：

使用 NodeSource 仓库安装：

# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证安装
node --version
npm --version

步骤 2: 安装 Codex CLI

Windows（使用 WSL）：

💡 Windows 支持仍在实验阶段，建议在 WSL 中运行

npm install -g @openai/codex

macOS：

使用 Homebrew（推荐）：

brew install --cask codex

Linux：

使用 npm 全局安装：

npm install -g @openai/codex

验证安装：

codex --version

显示版本号即表示安装成功

步骤 3: 配置认证

首次运行自动引导认证：

codex

推荐方式：使用 ChatGPT 账户登录

  • 支持账户类型：ChatGPT Plus、Pro、Business、Edu 或 Enterprise
  • 在提示时选择 "Sign in with ChatGPT"
  • 按照浏览器提示完成 OAuth 认证

📝 使用 API Key 认证（可选）：

如需使用 API Key，设置环境变量后使用以下命令：

# 设置环境变量
export OPENAI_API_KEY="YOUR_API_KEY"

# 使用 API Key 登录
echo $OPENAI_API_KEY | codex login --with-api-key

配置中转 API Base URL（可选）：

编辑配置文件 ~/.codex/config.toml：

preferred_auth_method = "apikey"

[model_providers]
[model_providers.openai]
base_url = "{{OPENAI_API_URL}}"
  • API Base URL: {{OPENAI_API_URL}}
  • 配置文件位置：~/.codex/config.toml
  • 需要先设置 OPENAI_API_KEY 环境变量

步骤 4: 开始使用

启动 Codex CLI：

codex

或使用自然语言命令：

codex "list all files in current directory"

💡 Codex CLI 可以将自然语言转换为对应的 Bash/ZShell/PowerShell 命令
