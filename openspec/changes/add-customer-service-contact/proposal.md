# Proposal: 添加客户服务联系功能

## Change ID
`add-customer-service-contact`

## Overview
为用户提供便捷的客户服务联系方式,包括在页面顶部增加客服按钮以及在钱包充值页面显示客服和闲鱼店铺二维码。管理员可在后台配置上传这些二维码图片。

## Motivation
当前系统缺乏明显的客户服务联系入口,用户遇到问题时难以快速找到帮助渠道。通过在关键位置展示客服联系方式,可以:
- 提升用户体验,降低用户获取帮助的难度
- 增加客服渠道的可见性,提高用户咨询率
- 为管理员提供灵活配置客服渠道的能力
- 在充值页面展示闲鱼店铺,提供额外的购买渠道

## Proposed Changes

### 1. 页面顶部客服按钮 (customer-service-header-button)
- 在页面右上角导航栏增加"联系客服"按钮
- 鼠标悬浮或点击时,以浮层形式显示管理员配置的客服二维码
- 支持图片缩放和预览功能
- 未配置二维码时,按钮隐藏或显示提示信息

### 2. 钱包页面二维码展示 (customer-service-wallet-display)
- 在钱包管理页面的账户充值区域下方,增加二维码展示区域
- 并排显示两个二维码:
  - 客服二维码 (与顶部按钮使用相同配置)
  - 闲鱼店铺二维码
- 每个二维码包含标题说明和图片
- 未配置时该区域不显示

### 3. 管理后台配置 (Backend & Admin UI)
- 在管理后台"运营设置 > 通用设置"中增加两个图片上传字段:
  - `CustomerServiceQRCode`: 客服二维码图片 URL
  - `XianyuQRCode`: 闲鱼店铺二维码图片 URL
- 支持图片预览和删除功能
- 图片存储建议使用现有的图片上传机制

## Success Criteria
- 页面顶部客服按钮正常显示和交互
- 鼠标悬浮/点击能正确展示二维码浮层
- 钱包页面二维码区域布局合理,响应式适配
- 管理后台能成功上传、预览和更新二维码图片
- 未配置二维码时界面优雅降级,不显示空白区域
- 所有文案支持中英文国际化

## Dependencies
- 依赖现有的 Option 配置系统 (model/option.go, controller/option.go)
- 可能需要复用现有的图片上传功能 (ImageUploadPermission)
- 前端需要使用 Semi Design 的 Popover/Modal 组件展示二维码

## Risks & Mitigations
- **风险**: 图片上传可能需要新的上传接口
  - **缓解**: 可以直接使用图片 URL 配置,管理员手动上传图片到图床

- **风险**: 移动端适配问题
  - **缓解**: 采用响应式布局,移动端优化交互方式 (点击代替悬浮)

## Related Changes
- 参考现有的 `WeChatAccountQRCodeImageURL` 配置实现
- 参考现有的 `TopUpLink` 和 `XianyuShopLink` 配置字段

## Alternatives Considered
1. **使用文本链接而非二维码**: 不够直观,二维码扫描更便捷
2. **仅在钱包页面显示**: 缺少全局入口,用户发现成本高
3. **使用第三方客服插件**: 增加系统复杂度,配置不灵活
