## 🎯 环境要求

### 必需软件

| 软件 | 最低版本 | 推荐版本 | 用途 |
|------|---------|---------|------|
| **Go** | 1.25.1 | 1.25.3+ | 后端编译运行 |
| **Node.js** | 18.0.0 | 20.0.0+ | 前端构建 |
| **Bun** | 1.0.0 | 最新版 | 前端包管理（推荐） |
| **PostgreSQL** | 12+ | 15+ | 数据库（远程或本地） |
| **Redis** | 6.0+ | 7.0+ | 缓存（本地） |

### 可选软件

- **npm/yarn** - 如果不使用 Bun
- **Make** - 使用 Makefile 构建（可选）

## 🚀 快速开始（5分钟）

### 前提条件检查

```bash
# 1. 检查 Go 版本
go version
# 应该显示: go version go1.25.x

# 2. 检查 Node.js 版本
node --version
# 应该显示: v18.x.x 或更高

# 3. 检查 Bun（推荐）
bun --version
# 应该显示: 1.x.x

# 4. 检查本地 Redis
redis-cli ping
# 应该返回: PONG

# 5. 测试远程 PostgreSQL 连接
psql -h 47.104.195.67 -U root -d new-api -p 5432
# 输入密码后应能成功连接
```

### 一键启动脚本

```bash
# 克隆代码（如果还没有）
git clone <your-repo-url>
cd new-api

# 运行一键启动脚本
./quick-start.sh
```

如果没有一键脚本，请按照下面的详细步骤操作。

## 📝 详细步骤

### 第一步：准备环境

#### 1.1 安装 Go

```bash
# Linux
wget https://go.dev/dl/go1.25.3.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.25.3.linux-amd64.tar.gz
export PATH=$PATH:/usr/local/go/bin

# macOS (Homebrew)
brew install go

# Windows
# 下载安装包: https://go.dev/dl/
```

#### 1.2 安装 Node.js 和 Bun

```bash
# 安装 Node.js (使用 nvm 推荐)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 20
nvm use 20

# 安装 Bun (推荐，速度更快)
curl -fsSL https://bun.sh/install | bash

# 或使用 npm
npm install -g bun
```

#### 1.3 启动本地 Redis

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install redis-server
sudo systemctl start redis
sudo systemctl enable redis

# macOS
brew install redis
brew services start redis

# 验证 Redis
redis-cli ping
# 应返回: PONG
```

#### 1.4 确保远程 PostgreSQL 可访问

```bash
# 测试连接
psql -h 47.104.195.67 -U root -d new-api -p 5432

# 如果连接失败，检查：
# 1. 防火墙是否开放 5432 端口
# 2. PostgreSQL 是否允许远程连接
# 3. 用户名密码是否正确
```

---

### 第二步：配置项目

#### 2.1 检查 .env 文件

项目根目录已有 `.env` 文件（如果没有，复制 `.env.local.example`）：

```bash
# 查看配置
cat .env

# 如果需要修改，编辑配置
vim .env
```

**重要配置项：**

```bash
# PostgreSQL - 远程数据库
SQL_DSN=postgresql://root:你的密码@47.104.195.67:5432/new-api

# Redis - 本地缓存
REDIS_CONN_STRING=redis://localhost:6379

# 应用端口
PORT=3000

# 调试模式
DEBUG=true
```

#### 2.2 修改数据库密码

**⚠️ 重要：** 将 `SQL_DSN` 中的密码改为实际密码：

```bash
# 编辑 .env 文件
vim .env

# 找到这行并修改密码
SQL_DSN=postgresql://root:实际密码@47.104.195.67:5432/new-api
```

---

### 第三步：安装依赖

#### 3.1 安装 Go 依赖

```bash
# 在项目根目录执行
go mod download

# 验证依赖
go mod verify
```

#### 3.2 安装前端依赖

```bash
# 进入 web 目录
cd web

# 使用 Bun 安装（推荐，速度快）
bun install

# 或使用 npm
npm install

# 返回根目录
cd ..
```

---

### 第四步：构建前端

前端必须先构建，因为后端通过 `embed.FS` 嵌入前端资源。

```bash
# 进入 web 目录
cd web

# 构建生产版本
bun run build

# 或使用 npm
npm run build

# 检查构建产物
ls -la dist/
# 应该看到 index.html, assets/ 等文件

# 返回根目录
cd ..
```

**构建成功标志：**
- `web/dist/` 目录包含完整的前端资源
- `web/dist/index.html` 文件存在
- `web/dist/assets/` 目录包含 JS/CSS 文件

---

### 第五步：运行项目

#### 方式一：直接运行（推荐开发）

```bash
# 在项目根目录执行
go run main.go
```

#### 方式二：编译后运行

```bash
# 编译
go build -o new-api main.go

# 运行
./new-api

# Windows
new-api.exe
```

#### 方式三：使用 air 热重载（开发推荐）

```bash
# 安装 air
go install github.com/air-verse/air@latest

# 运行
air
```

**首次访问：**

2. 默认管理员账号：`root`

```bash

curl http://localhost:3000/api/status

# 应该返回
{"success":true,"message":"..."}
```

## 🔧 开发模式

### 前端开发（热重载）

如果需要修改前端代码：

```bash
# 终端 1：启动后端
go run main.go

# 终端 2：启动前端开发服务器
cd web
bun run dev
# 或
npm run dev

# 前端开发服务器运行在 http://localhost:5173
# API 请求会代理到后端 http://localhost:3000
```

### 后端开发（热重载）

使用 `air` 实现后端热重载：

```bash
# 安装 air
go install github.com/air-verse/air@latest

# 创建 .air.toml 配置（可选）
air init

# 启动
air

# 修改 Go 代码后会自动重新编译和运行
```

## ❓ 常见问题

### 1. 前端构建失败

**错误：** `Error: Cannot find module 'xxx'`

**解决：**
```bash
cd web
rm -rf node_modules
bun install
bun run build
```

### 2. Go 依赖下载失败

**错误：** `go: module xxx: Get "https://proxy.golang.org/...": dial tcp: i/o timeout`

**解决：**
```bash
# 配置 Go 代理
go env -w GOPROXY=https://goproxy.cn,direct
go env -w GO111MODULE=on

# 重新下载
go mod download
```

### 3. Redis 连接失败

**错误：** `Could not connect to Redis at localhost:6379: Connection refused`

**解决：**
```bash
# 检查 Redis 是否运行
redis-cli ping

# 启动 Redis
# Linux
sudo systemctl start redis

# macOS
brew services start redis

# 手动启动
redis-server
```

### 4. PostgreSQL 连接失败

**错误：** `dial tcp 47.104.195.67:5432: connect: connection refused`

**解决：**
1. 检查服务器防火墙是否开放 5432 端口
2. 检查 PostgreSQL 配置允许远程连接
3. 验证用户名密码是否正确

```bash
# 测试连接
psql -h 47.104.195.67 -U root -d new-api -p 5432

# 如果需要，检查服务器端配置
# 编辑 postgresql.conf
listen_addresses = '*'

# 编辑 pg_hba.conf
host    all             all             0.0.0.0/0               md5
```

### 5. 端口被占用

**错误：** `bind: address already in use`

**解决：**
```bash
# 查看占用端口的进程
lsof -i :3000
# 或
netstat -tuln | grep 3000

# 杀死进程
kill -9 <PID>

# 或修改 .env 中的端口
PORT=3001
```

### 6. 前端页面空白

**可能原因：**
- 前端未构建或构建不完整
- embed.FS 未正确加载资源

**解决：**
```bash
# 重新构建前端
cd web
rm -rf dist
bun run build

# 确保 dist 目录有内容
ls -la dist/

# 重新编译后端
cd ..
go build -o new-api main.go
./new-api
```

### 7. 数据库迁移失败

**错误：** `Error 1054: Unknown column 'xxx' in 'field list'`

**解决：**
```bash
# 检查数据库结构
psql -h 47.104.195.67 -U root -d new-api -c "\d+ channels"

# 手动运行迁移
# 查看 model/*.go 中的数据库结构
# 如需要，手动执行 ALTER TABLE 语句
```

## 🛠️ 构建生产版本

### 编译优化版本

```bash
# 构建前端
cd web
bun run build
cd ..

# 编译后端（优化版本）
go build -ldflags "-s -w" -o new-api main.go

# 运行
./new-api
```

### 交叉编译

```bash
# Linux 64位
GOOS=linux GOARCH=amd64 go build -ldflags "-s -w" -o new-api-linux main.go

# Windows 64位
GOOS=windows GOARCH=amd64 go build -ldflags "-s -w" -o new-api-windows.exe main.go

# macOS 64位
GOOS=darwin GOARCH=amd64 go build -ldflags "-s -w" -o new-api-macos main.go

# macOS ARM (M1/M2)
GOOS=darwin GOARCH=arm64 go build -ldflags "-s -w" -o new-api-macos-arm64 main.go
```
